# 知更四功能验收指南

按顺序逐个功能验收，每个功能先看能不能跑通，再看数据对不对。

---

## 验收前准备

1. 启动应用：`npm run dev`
2. 确认设置页 API Key 已配置且连接测试通过
3. 创建至少 2 个创作者账号（比如"测试B站UP"和"测试抖音博主"）
4. 确认当前激活的是其中一个

---

## 功能一：内容日历

### 基础流程

1. 侧边栏点击"日历" → 进入日历页面
2. 点击"+ 新建排期" → 弹出表单
3. 选平台"B站"，填标题"测试视频"，选明天，填时间"20:00"
4. 保存 → 日历上明天的格子出现一个蓝色色点 + 标题
5. 点击那个排期 → 可以编辑/删除
6. 改状态为"已发布" → 状态文字变了

### 隔离验证

1. 切换到另一个创作者
2. 日历应该为空（看不到第一个创作者的排期）

### 选题联动

1. 去灵感库，保存一个选题
2. 回到选题卡片，点"排期"按钮
3. 回到日历 → 应该多了一条排期，标题自动填充

### 通过标准

- [ ] 日历页面正常渲染，无白屏
- [ ] 新建/编辑/删除排期正常
- [ ] 平台色点正确显示
- [ ] 切换创作者后数据隔离
- [ ] 控制台无报错

---

## 功能二：评论舆情

### 基础流程

1. 先去数据概览，爬取一个B站创作者（比如 UID 440609243）
2. 在视频列表中点击一个视频 → 进入详情面板
3. 找到"评论分析"区域 → 点击"爬取评论"
4. 等待完成 → 应该显示评论列表
5. 点击"AI 分析" → 等待完成 → 显示情感分布 + 热门话题
6. 点击"写入记忆" → 提示成功

### 数据验证

1. 打开 SQLite 查看（或在 Agent 对话中说"查看我的记忆"）
2. 确认 memories 表新增了"观众反馈关注点"类型的记忆

### 隔离验证

1. 切换到另一个创作者
2. 评论分析区域应该为空（没有评论数据）

### 通过标准

- [ ] 爬取评论不报错，有数据返回
- [ ] AI 分析后情感色标正确（绿/灰/红）
- [ ] 热门话题标签显示
- [ ] 写入记忆成功
- [ ] 切换创作者后数据隔离
- [ ] 控制台无报错

---

## 功能三：内容复盘

### 基础流程

1. 去数据概览 → 切到"复盘"Tab
2. 点击"+ 录入数据"
3. 填写：B站、"测试视频"、今天、播放 10000、点赞 500、评论 50、收藏 100
4. 保存 → 复盘列表出现这条记录，显示互动率
5. 点击"AI 分析" → 等待 → 显示分析文本
6. 确认 memories 表新增了"B站内容表现"记忆

### 数据验证

1. 播放量 10000，互动率 = (500+50+100)/10000 = 6.5%
2. 界面上应该显示"互动率 6.5%"
3. 如果填了预测播放量，应该显示偏差百分比

### 隔离验证

1. 切换创作者 → 复盘列表为空

### 通过标准

- [ ] 录入数据正常保存
- [ ] 互动率计算正确
- [ ] AI 分析生成复盘文本
- [ ] 写入 memories 成功
- [ ] 切换创作者后数据隔离
- [ ] 控制台无报错

---

## 功能四：竞品订阅

### 基础流程

1. 去灵感库 → 切到"竞品追踪"Tab
2. 点击"+ 订阅新竞品" → 输入 B站 UID（比如 440609243）
3. 保存 → 竞品列表出现
4. 点击"立即检查" → 等待完成
5. 如果有新内容 → 通知区域出现红色未读通知
6. 点击"标记已读" → 红点消失

### 通知验证

1. 侧边栏应该显示未读通知红点（如果有的话）
2. 点击"全部标记已读" → 所有通知变灰

### 隔离验证

1. 切换创作者 → 竞品列表和通知都为空

### 通过标准

- [ ] 订阅竞品正常
- [ ] 立即检查不报错
- [ ] 新内容通知正确显示
- [ ] 标记已读功能正常
- [ ] 侧边栏红点正确
- [ ] 切换创作者后数据隔离
- [ ] 控制台无报错

## 快速验收脚本（可选）

如果想自动化跑一遍基础检查，在 DevTools Console 里粘贴：

```javascript
// 快速检查所有新 API 是否暴露
const apis = [
  'scheduleCreate', 'scheduleList', 'scheduleUpdate', 'scheduleDelete', 'scheduleRecommendedTimes',
  'commentsCrawl', 'commentsList', 'commentsAnalyze', 'commentsSummary', 'commentsWriteMemories',
  'reviewCreate', 'reviewList', 'reviewUpdate', 'reviewDelete', 'reviewStats', 'reviewAiAnalyze',
  'competitorCheckNow', 'competitorAlerts', 'competitorAlertRead', 'competitorAlertReadAll',
]
const missing = apis.filter(a => typeof window.electronAPI[a] !== 'function')
if (missing.length) {
  console.error('❌ 缺失 API:', missing)
} else {
  console.log('✅ 全部 20 个新 API 已暴露')
}
```

## 验收顺序

建议按这个顺序，每个功能验收通过后再进下一个：

```
日历（最简单） → 复盘（纯 CRUD） → 评论（依赖爬虫） → 竞品（依赖定时任务）
```

如果某个功能卡住了，先记下来，不影响其他功能的验收。
